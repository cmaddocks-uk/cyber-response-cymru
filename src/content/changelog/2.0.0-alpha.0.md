The Welsh fork's first proper release. Carries every architectural improvement the English tool accumulated between v1.7.0 and v2.6.5, plus a deliberate Welsh-context framework sweep on top.

## Architecture port (v0.1 → v2.x)

- Single-file `index.html` rebuilt onto Astro 5 + React 19 + Tailwind 3 — same shape as the English tool at v2.6.5. Future bug fixes and feature work can now be forward-ported between the two tools symmetrically.
- 11 Astro pages, 7 page-level React islands (Readiness, Plan Builder, Output, Action Plan, First 30, Governor, Tabletop), a full Word `.docx` document model, and Zustand-style state with per-page selectors.
- Strict Content Security Policy (`default-src 'none'`) with `require-trusted-types-for 'script'` + a default Trusted Types policy enforced in Chrome and Edge.
- Static GitHub Pages deployment via the `.github/workflows/deploy.yml` workflow inherited from the English line.
- v0.1.0 single-file preserved at the repo root as `*.v0.1.backup` files (`index.html.v0.1.backup`, README, SECURITY, 404, robots, etc.) for archival reference.

## Welsh-context framework swap

- `Framework` type renamed: `DfE` → `WelshGov`, `RPA` → `LAcover`, `Ofsted` → `Estyn` (with framework metadata pointing at Cyber Resilient Wales, LA cyber cover guidance, and Estyn inspection arrangements).
- New plan field `external.welshGovContact` (Welsh Government / consortium digital education contact) replacing the English `dfeContact` semantic slot.
- New plan field `external.rocuCyberProtect` for the regional ROCU Cyber PROTECT team (TARIAN for South Wales, Gwent, Dyfed-Powys; NWROCU for North Wales).
- 16 readiness questions reworded to reference NCSC, Welsh Gov / Cyber Resilient Wales, LA cyber cover, Estyn and CAF where they previously named DfE Standards, RPA, Ofsted or the DfE Academy Trust Handbook.
- 8th default asset register row added for **Hwb-hosted services** (`hwb@gov.wales` as the incident contact). Row 1 "SEN" → "ALN/ASN" (Welsh ALN Code).
- 9 playbooks (ransomware, data breach, account compromise, phishing, BEC, AI-driven extortion, denial of service, insider threat, SaaS supplier incident). BEC and AI-extortion `whyMatters` written from scratch with Welsh-context routing through TARIAN/NWROCU and the LA cyber lead. SaaS playbook title and notification step explicitly list Hwb services.
- 6 tabletop scenarios — source labels swapped from DfE Cyber Security Hub to NCSC's schools section; Scenario F (cloud / ransomcloud) reworked to include Hwb services and `hwb@gov.wales`.
- Communications templates (parent letter, governor briefing, ICO Article 33 draft) and the governor-question set updated for Welsh assurance language.
- First 30 Minutes card lists the TARIAN / NWROCU split, drops English-specific phone numbers, replaces "DfE Academy Trust Handbook" prohibition with the NCSC + Welsh Government advisory.
- Document-model layer (Word `.docx` export pipeline) swept for the same DfE/RPA/Ofsted references so the downloaded plans match the on-screen content.

## Brand system

- CIRP brand mark (the half-shaded "30" disc + monolithic chip favicon) inherited from the English tool's v2.6.0–2.6.5 brand work.
- Welsh descriptor under the wordmark: `Cynllun Ymateb i Achosion Seiber` in the home-page footer (Lockup A) and About-page hero (Lockup D). Wordmark stays as `CIRP™` to keep brand consistency across both tools.
- Open Graph image regenerated at 1200×630 with a Welsh frameworks strip: `NCSC · LLYWODRAETH CYMRU · ESTYN · TARIAN · ICO · CYBER ESSENTIALS · HWB`.
- "Wales has no DfE-style sector cyber hub — this tool exists to fill that gap" callout replaces the English DfE Cyber Security Hub link on FrameworksPanel and ScenarioPicker.
