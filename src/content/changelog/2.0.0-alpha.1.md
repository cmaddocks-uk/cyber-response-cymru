First batch of bug fixes forward-ported from the English sister tool now that both tools share the v2.x Astro architecture.

## Multi-line text fields render correctly (from English v2.6.6)

- Any multi-line text entered into the Plan Builder (playbook school-specific notes, parent-letter templates, ICO Article 33 notifications, account-compromise notes, etc.) was rendering as a single run-on paragraph with all line breaks collapsed into spaces. Both the on-screen view and the Word `.docx` export were affected.
- `DocumentRenderer.tsx` paragraph blocks now use `whiteSpace: 'pre-wrap'` so the browser preserves newlines from user-entered content.
- `word.ts` paragraph blocks now split on double-newlines into separate Word paragraphs and use `TextRun({ break: 1 })` for single newlines within — so structured "Label: value" content reads as it was typed.
- No schema changes, no data migration needed. Existing saved plans render correctly without modification.

## Changelog page redesign (from English v2.6.7)

- Versions now sort by date descending with a true numeric semantic-version tiebreak within each date (so `2.6.10` would correctly sit above `2.6.2` when it ships).
- Releases grouped by date — the date appears once as a small uppercase heading above the cluster of releases for that day.
- Release-type badges added — small uppercase pills next to each version: **Feature** (blue), **Fix** (green), **Security** (red), **UI** (purple), **Breaking** (amber). Optional `tags: ['Feature' | 'Fix' | 'Security' | 'UI' | 'Breaking']` field added to the changelog frontmatter schema.
- One-line summary visible on collapsed rows (on screens ≥ 640 px) — you can scan what each release was about without expanding it.
- Chevron icon replaces the static caret — rotates smoothly when the entry is expanded.
- "Latest" pill on the newest release for at-a-glance currency.
- Native `<details>` / `<summary>` semantics preserved for keyboard navigation and focus rings.
